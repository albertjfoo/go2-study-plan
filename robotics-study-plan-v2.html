Project Files — What Each One Does
config/robot.env
Settings file. Two variables: which network port to use, and the robot's IP address. Change ROBOT_NETWORK_INTERFACE if you switch between ethernet and WiFi.
requirements.txt
Shopping list for Python packages. Run pip install -r requirements.txt and Python fetches everything the project needs.
PackageWhy you need it
unitree_sdk2pyUnitree's SDK — subscribers, clients, message types
cycloneddsDDS middleware the robot uses to communicate
numpyArray math; SDK returns positions/velocities as arrays
python-dotenvReads robot.env into the environment
richPretty terminal tables and colors
src/go2_connection.py
A module imported by every script. Call init_channel() once at startup before anything else. It reads the network config, writes a CycloneDDS XML config (workaround for a buffer overflow bug in the SDK), then starts the DDS communication layer.
scripts/connect_test.py
Subscribes to rt/lf/sportmodestate and prints a live table of position, velocity, and gait data. Only updates when the robot is in sport mode — enable via the Unitree app.
scripts/imu_live.py
Subscribes to rt/lowstate and prints accelerometer, gyroscope, and roll/pitch/yaw live. Also logs every reading to logs/imu_<timestamp>.csv. Works regardless of robot mode — always publishing.
